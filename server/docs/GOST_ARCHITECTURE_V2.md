# GOST 架构 V2 - 完整重新设计

## 一、核心设计原则

1. **端口分组** - 每个商户占用 6 个端口（3 个 TLS + 3 个 TCP）
2. **双模式并存** - 同时支持 TLS 加密和 TCP 直连，手机端可切换
3. **层级清晰** - 系统服务器 → 商户GOST → 业务程序
4. **一键操作** - 创建/删除/换IP 全自动完成

## 二、端口规划

### 2.1 商户端口分配（系统服务器监听）

每个商户分配一个基础端口 `BASE_PORT`，占用 6 个端口：

```
┌────────────────────────────────────────────────────────────────┐
│  商户基础端口: 20000                                           │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  TLS 加密模式（手机端配置 TLS 时使用）                          │
│  ├─ 20000 (TCP over TLS)  → 商户服务器:10010 → 业务:10000     │
│  ├─ 20001 (WS over TLS)   → 商户服务器:10011 → 业务:10001     │
│  └─ 20002 (HTTP over TLS) → 商户服务器:10012 → 业务:10002     │
│                                                                │
│  TCP 直连模式（手机端配置 TCP 时使用）                          │
│  ├─ 20003 (TCP 直连)      → 商户服务器:10000 (跳过GOST)       │
│  ├─ 20004 (WS 直连)       → 商户服务器:10001 (跳过GOST)       │
│  └─ 20005 (HTTP 直连)     → 商户服务器:10002 (跳过GOST)       │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### 2.2 商户服务器固定端口

商户服务器上的 GOST 固定监听：

```
┌────────────────────────────────────────────────────────────────┐
│  商户服务器 GOST（relay+tls 监听）                              │
│  ├─ 10010  →  转发到 127.0.0.1:10000 (业务程序 TCP)           │
│  ├─ 10011  →  转发到 127.0.0.1:10001 (业务程序 WS)            │
│  └─ 10012  →  转发到 127.0.0.1:10002 (业务程序 HTTP)          │
│                                                                │
│  业务程序监听（固定端口）                                       │
│  ├─ 10000  TCP 协议                                           │
│  ├─ 10001  WebSocket 协议                                     │
│  └─ 10002  HTTP 协议                                          │
└────────────────────────────────────────────────────────────────┘
```

### 2.3 端口占用规则

- 每个商户占用 **6 个连续端口**
- 端口范围建议：20000-60000
- 创建商户时检查 `port` 到 `port+5` 是否可用

## 三、数据流向

### 3.1 TLS 加密模式（推荐）

```
手机客户端
  │ 连接到 系统服务器:20000 (TLS)
  ▼
系统服务器 GOST (tcp-tls-20000)
  │ handler: tcp
  │ chain: relay+tls → 商户服务器:10010
  ▼
商户服务器 GOST (local-tcp-10010)
  │ listener: tls
  │ handler: relay
  │ forwarder → 127.0.0.1:10000
  ▼
业务程序 :10000
```

**特点：**
- 全链路 TLS 加密
- 安全性高
- 性能略低（加密开销）

### 3.2 TCP 直连模式

```
手机客户端
  │ 连接到 系统服务器:20003 (TCP)
  ▼
系统服务器 GOST (tcp-direct-20003)
  │ handler: tcp
  │ forwarder → 商户服务器:10000 (跳过商户GOST)
  ▼
业务程序 :10000
```

**特点：**
- 无加密
- 性能最高
- 适合内网或已有其他加密层的场景

### 3.3 手机端切换逻辑

```dart
// 手机端配置示例
class ServerConfig {
  String serverIP;      // 系统服务器 IP
  int basePort;         // 商户基础端口，如 20000
  bool useTLS;          // 是否使用 TLS

  int get tcpPort => useTLS ? basePort : basePort + 3;
  int get wsPort => useTLS ? basePort + 1 : basePort + 4;
  int get httpPort => useTLS ? basePort + 2 : basePort + 5;
}
```

## 五、一键操作流程

### 5.1 创建商户（一键部署）

```
API: POST /merchant
输入: { name, port, serverIP, ... }

执行流程:
┌─────────────────────────────────────────────────────────────────┐
│ 1. 验证端口可用性（检查 port 到 port+5）                        │
│                                                                  │
│ 2. 创建数据库记录                                               │
│    ├─ merchants 表                                              │
│    └─ servers 表（商户服务器，type=1）                          │
│                                                                  │
│ 3. 配置商户服务器 GOST（同步等待完成）                          │
│    └─ 创建 local-tcp-10010, local-ws-10011, local-http-10012   │
│                                                                  │
│ 4. 配置所有系统服务器 GOST（异步队列）                          │
│    └─ 对每个系统服务器:                                         │
│       ├─ 创建 TLS 转发: tcp-tls-{port}, ws-tls-{port+1}...     │
│       └─ 创建直连转发: tcp-direct-{port+3}, ws-direct-{port+4}. │
│                                                                  │
│ 5. 返回结果（包含配置状态）                                      │
└─────────────────────────────────────────────────────────────────┘
```

## 九、错误处理

### GOST 配置失败处理

1. **创建商户时 GOST 配置失败**
   - 回滚已创建的服务
   - 返回错误信息
   - 商户状态设为「配置中」

2. **创建系统服务器时 GOST 配置失败**
   - 单个商户失败不影响其他商户
   - 记录失败的商户 ID
   - 提供重试接口

3. **换 IP 时 GOST 配置失败**
   - 数据库已更新，GOST 配置失败
   - 提供手动重试接口
   - 记录详细错误日志
